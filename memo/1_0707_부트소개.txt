- 개발자가 이용을 많이 하면서, 서브 -> 메인으로 자리 잡음.
- 스프링 -> 부트를 의미함.
- 자동설정.
- 테스트롤 설정 없이 시작 했지만, 보면, 드라이버,

해서 실행이 안될 뿐이고, 서버가 내장이 되어 있다. (톰캣서버 기본 탑재)
- 배포 할 때, 내장 서버가 포함이 되어서, jar 압축파일로 배포시 편하다.
- xml 설정, root-context, servlet-context. log4j2.xml , 많이 생략이 됨.

- 레거시 마이바티스 프레임워크를 이용했고, 부트에서도 물론이 이용이 가능하고,
- 부트에서는 JPA 이용해서, ORM 기술을 이용해서, 데이터베이스를 처리할 예정.

스프링 부트(Spring Boot)는 스프링 프레임워크(Spring Framework)를 기반으로 한 오픈 소스 자바 프레임워크로, 복잡한 스프링 환경 설정을 크게 줄이고, 빠르고 간편하게 독립 실행형(standalone) 애플리케이션을 개발할 수 있도록 도와줍니다.

### 스프링 부트의 특징

1. **자동 구성(Auto Configuration)**
   - 스프링 부트는 다양한 설정을 자동으로 해주어 개발자가 필요한 최소한의 설정만 하면 됨.
   - 예를 들어 데이터베이스, 웹 서버(Tomcat 등), JPA 등 여러 컴포넌트를 자동으로 감지하고 설정함.

2. **내장 서버(Embedded Server) 지원**
   - Tomcat, Jetty, Undertow 등 웹 서버를 내장하고 있어 따로 WAS(Web Application Server)를 설치할 필요 없이 JAR 파일만 실행하면 바로 구동 가능.

3. **독립 실행형 애플리케이션(Standalone Application)**
   - 별도의 웹서버 배포 없이 `java -jar` 명령어로 바로 실행할 수 있음.

4. **의존성 관리(Dependency Management)**
   - 스타터(Starter) 의존성을 통해 필요한 의존성들을 한 번에 쉽게 추가할 수 있도록 도와줌.
   - 예: `spring-boot-starter-web`, `spring-boot-starter-data-jpa` 등.

5. **생산성 향상(Developer Productivity)**
   - 다양한 개발 도구, 라이브 리로드(DevTools), 쉬운 테스트 환경 등을 제공하여 개발 생산성을 높여줌.

6. **의미 있는 기본값(Opinionated Defaults)**
   - 스프링 부트는 일반적으로 많이 쓰이는 설정을 기본값으로 제공하여, 초보자도 쉽게 시작할 수 있음.

7. **외부 설정(Externalized Configuration)**
   - `application.properties` 또는 `application.yml` 파일을 통해 환경설정을 손쉽게 관리할 수 있음.

### 스프링 부트의 소개

스프링 부트는 "Convention over Configuration"(설정보다 관례 우선) 철학에 따라, 복잡한 XML 기반의 설정이나 수많은 설정 파일의 부담 없이 간단하게 스프링 기반 애플리케이션을 만들 수 있도록 설계되었습니다.

기업용 대규모 서비스부터 스타트업의 프로토타입 개발까지 폭넓게 사용되며, 마이크로서비스 아키텍처(MSA)에서도 많이 활용됩니다. 스프링 부트는 표준 스프링 프레임워크의 강력함을 유지하면서, 빠른 개발과 배포, 유지보수를 가능하게 하는 현대적인 자바 개발의 핵심 도구입니다.

---

**정리**
스프링 부트는 빠르고 쉽게 스프링 애플리케이션을 만들고 관리할 수 있도록 도와주는 프레임워크입니다. 자동 구성, 내장 서버, 스타터 의존성 같은 기능 덕분에 개발자의 편의성과 생산성이 크게 향상됩니다.
